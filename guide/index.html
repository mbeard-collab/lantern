User Guide

Everything you can do on this site, in plain English. Sections labelled everyone work for any signed-in user; admin sections require admin sign-in.

Quick start — create or update a dashboard

  1. Go to /studio/ and sign in with your @govspend.com email and the shared admin password (ask Matt for it).
  2. Have data? Stay on the Build with Claude tab and drop your .csv/.xlsx/.json. Already have a finished page? Switch to Upload HTML and drop the .html (plus any data files it loads).
  3. For Target: pick the existing dashboard to update it, or + New dashboard and type a slug (see below).
  4. Click Ship to site → and confirm. It's live in about 30 seconds.
The slug is the dashboard's URL name — it must be "kebab-case." Lowercase letters, numbers, and hyphens only; no spaces, capitals, or symbols.
✅ q3-pipeline   ai-spend-2026   renewals
✗ Q3 Pipeline   ai_spend   Renewals!
The admin password is one shared secret for all admins. Don't paste it into Slack, email, or screenshots.

1. Viewing dashboards everyone

The main page (/) is the hub. Every dashboard is a card grouped by section.

  1. Go to the home page.
  2. Sections in the top nav: Technology Ops, Operational, Intelligence, AI & Cost, Reference. Click any to jump.
  3. Click any dashboard card to open it in a new tab.

2. Voting on a ballot everyone

No sign-in required for voting — anyone who can open the site can vote.

  1. Click Voting in the top nav (or go to /voting/).
  2. Pick a ballot from the list, click Open ballot.
  3. Type your name and your @govspend.com email at the top. Your email is how the system remembers your vote.
  4. For each item, click 1 (least important) through 5 (critical), or Skip if you have no opinion.
  5. When you're done, click Submit ballot. You'll see a "Submitted [date]" confirmation in the header.
Want to change your votes later? Just revisit the ballot URL, update your votes, and submit again. The system uses your email as the key, so the latest submission wins. Your previous votes are restored automatically when you re-enter the same email.

3. Viewing voting results everyone

Aggregated results (average score, vote distribution) are visible to everyone. Individual ballots are admin-only.

  1. From /voting/, click View results on any ballot.
  2. Items are sorted by average score (highest priority on top).
  3. The colored bars show how many people voted 1, 2, 3, 4, or 5 on each item.
  4. Click ↻ Refresh to pull the latest numbers without reloading.

To see who voted what, an admin can click Show individual ballots and sign in.

4. Signing in admin

Most admin features (Studio, ballot management, raw votes) ask you to sign in once. The sign-in is remembered on your device until you sign out.

  1. Go to /studio/ (or click an admin-only action like "Update items" on a ballot).
  2. Enter your @govspend.com email and the admin password (ask Matt if you don't have it).
  3. You'll see your email in a green pill at the top right when you're signed in. Click the × next to it to sign out.
The admin password is a single shared secret. Don't paste it into Slack, email, or screenshots.

5. Creating a new dashboard admin

Two flows. Use whichever fits the moment.

Option A — Chat with Claude (start from data)

  1. Go to /studio/, sign in if you haven't.
  2. Make sure the Build with Claude tab is selected.
  3. Drag a data file in: .csv, .xlsx, or .json. Multiple files are fine.
  4. Pick a target: + New dashboard (and type a slug like q3-pipeline), or pick an existing dashboard to overwrite.
  5. Click Start →. The right pane fills in as Claude works.
  6. Iterate via the chat: "make the main chart a bar", "drop the second KPI", "use orange for the at-risk pill". Claude regenerates the dashboard each turn.
  7. When happy, click Ship to site → in the top nav. Confirm. The dashboard is live within ~30 seconds.

Option B — Upload finished HTML

  1. In Studio, click the Upload HTML tab.
  2. Drop your .html file. A preview appears below.
  3. Pick a target (new or existing dashboard).
  4. Click Ship to site →. Same confirmation flow.
For new dashboards, Studio asks you a few questions (title, section, owner, cadence) so it can register the dashboard on the landing page. Sections are: tech-ops, operational, intelligence, ai-cost, reference.

6. Updating an existing dashboard admin

Same flow as creating one — just pick the existing dashboard from the target dropdown.

  1. Go to /studio/.
  2. Pick your flow (chat or upload HTML).
  3. For target, choose the existing dashboard from the dropdown.
  4. Drop your file(s), iterate or preview, click Ship to site →.

The existing dashboard is replaced. Its registry entry on the home page stays the same.

7. Deleting a dashboard admin

  1. In Studio, click Manage in the top nav.
  2. The Dashboards tab lists every dashboard. Each row has an Open link and a red Delete button.
  3. Click Delete. Confirm in the popup.

The dashboard file is removed from the repo, the entry on the home page is removed, and a fresh deploy fires.

Deletion is two GitHub commits (one removes the file, one updates the registry). It can't be undone from this UI, but the file history is in GitHub if you ever need to recover one.

8. Refreshing a voting ballot admin

When the underlying Jira items change — items added, removed, status updated — replace the ballot's items.json with the latest export.

  1. Get a fresh items.json file (from the Jira export script the product team uses).
  2. In Studio, click Manage → switch to the Voting tab.
  3. Find the ballot, click Update items….
  4. Pick the new items.json file.
  5. Confirm. The file is committed and the ballot reflects the change within ~30 seconds.

Existing votes are preserved across refreshes. Voters who already submitted will see "Last submitted [date]" — they can return and re-submit to vote on any new items.

9. Adding a brand-new ballot admin

Right now this is a two-step manual action via GitHub. A "+ New ballot" button in Studio is on the roadmap.

  1. In the GitHub repo (mbeard-collab/lantern), add a new file at voting/<poll-id>/items.json (e.g. voting/q3-roadmap/items.json). Use voting/gs15687/items.json as a structure reference.
  2. Edit voting/polls.json to add an entry for your new poll: { id, title, description, status, opened_at }.
  3. Commit both changes. The ballot is live at /voting/<poll-id>/ within ~30 seconds, and it'll show up on the /voting/ list page.

If you'd rather Matt do it, send him the Jira export + the title and he can wire it up in 5 minutes.

10. Troubleshooting

"I clicked Ship and nothing happens" / button is greyed out

Look for a small hint line above the button. Common causes: no data file uploaded yet, no target dashboard picked, slug isn't lowercase/hyphens, or HTML isn't a complete document.

"The studio is blank / asks me to sign in every time"

You signed out (or your localStorage was cleared). Just sign in again. If your email isn't on the admin list, ask Matt to add you.

"My ballot won't submit"

Make sure both name and email fields are filled in. Email must include @. If it still won't go through, check the small toast at the bottom for the actual error.

"Claude generated a broken dashboard"

Tell it in chat — "the chart is empty, can you fix it?" or "make a simpler version with just the KPIs." It'll regenerate. If you're stuck, switch to Upload HTML mode and give it a hand-edited starting point.

Anything else

Ping Matt. Mention what you were doing and what you saw — screenshots help.
